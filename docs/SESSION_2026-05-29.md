# Session 2026-05-29: Encoding Fix + GET Decode Calibration

## 完了したこと

### 1. SET encoding 修正 (effect blocks)

**問題**: effect block の bipolar パラメータが `raw_float=True` で display value をそのまま送信していた。FM9 は全パラメータで normalized 0.0–1.0 を期待するため、20.0 を送ると clamp → 100% になっていた。

**修正**: 全 effect block パラメータを unified normalized encoding に統一。

```python
# Before (broken):
elif param_type == "bipolar":
    midi.set_param_value(block_id, pid, float(value), 1.0, raw_float=True)

# After (correct):
elif param_type == "bipolar":
    midi.set_param_value(block_id, pid, float(value), param_max)
```

**確定した SET encoding ルール:**

| パラメータタイプ | encoding | 式 |
|---|---|---|
| continuous | normalized | `value / max` |
| bipolar | normalized | `value / max` (FM9 が ±方向を解釈) |
| switch | raw_float | `0.0` or `1.0` |
| enum | raw_float | integer as float |
| signed_int | raw_float | semitone value |

旧コードの `NORMALIZED_SUFFIXES` heuristic (`_MIX`, `_LEVEL`, `_PAN`) は削除。全パラメータが同じ encoding を使う。

### 2. GET decode calibration システム構築

**問題**: SET と GET で異なる max 値を使っている。
- SET: `display_max` (cache 由来、UI 表示用) で normalize → FM9 が正しく解釈
- GET: raw 値は `internal_max` (firmware 内部の格納 range) に対する比率で格納

例: Delay Time — display_max=1000ms, internal_max=16000ms
- SET: `350/1000=0.35` → FM9 は 350ms と解釈 ✅
- GET: raw=1430 → `1430/65534*16000=349ms` ✅ (max=1000 で decode すると 21.8ms ❌)

**さらに bipolar パラメータに 2 種類の格納方式がある:**

| 格納方式 | raw=0 の意味 | decode 式 | 例 |
|---|---|---|---|
| center-offset | -100% (min) | `(raw-32767)/32767*max` | Feed, Feedl/r, Spread, Gain1/2 |
| zero-based | 0 (center) | `raw/65534*max` | Depth, Levell/r, Mstrfdbk, Killdry |

**解決**: `calibrate_decode.py` を作成。各パラメータに対して:
1. SET 0 → GET raw → center (raw≈32767) か zero (raw≈0) かを判定
2. SET 50% → GET raw → `decode_max` を逆算

結果を all_params.json に `decode_max` / `decode_style` フィールドとして記録。

## エンコーディングルール（最終確定版）

### SET (sub=0x09) — IEEE 754 float, 5×7bit packed

**全ブロック共通**: `set_param_value(block_id, pid, value, max)` で統一。

| ブロック | パラメータ | encoding |
|---------|-----------|----------|
| Amp/Drive | continuous (Gain, Bass, etc.) | `value / max` → 0.0-1.0 |
| Amp/Drive | bipolar (Level, Balance) | raw_float: display value directly |
| Amp/Drive | switch | raw_float: 0.0 or 1.0 |
| Effect blocks | continuous | `value / max` → 0.0-1.0 |
| Effect blocks | bipolar | `value / max` → ±1.0 (FM9 が方向を解釈) |
| Effect blocks | switch | raw_float: 0.0 or 1.0 |
| Effect blocks | enum | raw_float: integer index |
| Effect blocks | signed_int (Pitch Shift) | raw_float: semitone value |
| Cab | DynaCab R/Z | normalized 0.0-1.0 |
| Cab | everything else | raw_float |

### GET (func=0x1F) — 21bit integer, 3×7bit packed

| decode_style | 式 | 判定方法 |
|---|---|---|
| center | `(raw - 32767) / 32767 * decode_max` | SET 0 → raw≈32767 |
| zero (default) | `raw / 65534 * decode_max` | SET 0 → raw≈0 |
| uncalibrated bipolar | `raw / 65534 * (max-min) + min` | fallback |
| switch | `raw != 0` | — |
| enum | `raw` (integer) | — |

`decode_max` は calibration で実測。未 calibration の場合は `display_max` を使用 (不正確な場合あり)。
